### Optional: Vanity URL / Redirect (recommended for branding)

If you want product links to use a branded URL (for example `https://usbg.org/template`) you can configure a simple redirect on your web server or edge worker that forwards visitors to the Apps Script `/exec` URL. The project stores an optional `PUBLIC_WEBAPP_URL` (Script Property) which the Admin UI will prefer when generating "Copy Link" URLs.

Important: setting `PUBLIC_WEBAPP_URL` only changes the links generated by the admin UI ‚Äî it does not make Apps Script run on your domain. To make the vanity link usable you must configure a redirect or proxy on your domain.

Recommended: configure a redirect so the vanity link forwards to the Apps Script exec URL. This is the simplest and most reliable approach.

Nginx example (simple 302 redirect):

```nginx
location /template {
   return 302 https://script.google.com/macros/s/AKfy.../exec;
}
```

Cloudflare Worker (edge redirect) example:

```js
addEventListener('fetch', event => {
   const url = new URL(event.request.url);
   if (url.pathname === '/template') {
      return event.respondWith(Response.redirect('https://script.google.com/macros/s/AKfy.../exec', 302));
   }
   return event.respondWith(fetch(event.request));
});
```

How to use with this project:
- Deploy the web app (copy the `/exec` URL).
- Configure your server or edge rule to redirect the vanity path to the `/exec` URL.
- In the Admin Panel (‚öôÔ∏è Set Public URL) paste the vanity URL (for example `https://usbg.org/template`) into the prompt and save ‚Äî the app will validate the URL and warn if it doesn't appear to redirect to Apps Script.
- Now the Admin "Copy Link" button will generate links using the vanity URL.

Notes and caveats:
- The browser address bar will change to the Apps Script URL after the redirect. This is expected for a simple redirect. If you need the browser to retain `usbg.org` for the full session you must implement a domain-preserving reverse proxy (more complex; see docs above for tradeoffs).
- If you later change the Apps Script deployment URL, update the redirect target on your server and the `PUBLIC_WEBAPP_URL` if necessary.
